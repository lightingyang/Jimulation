# Jimulation 1.0.9.4 Changelog

**发布日期：** 2026-03-30

---

## 集装箱仿真

- 修复 `Config_DFICNB.yaml` 中 `总装` 设备的 `process_type`：`processing` -> `assembly`
- 修复后，装配树场景下子件全部完成后可以正常触发父节点总装流程

## 预处理开卷接口

- 请求参数调整：
  - `preprocessing_lead_time_minutes` 重命名为 `lead_time_minutes`
  - 新增可选字段 `scheduling_window_days`
  - 新增可选字段 `rest_days`，用于传入总排产区间内希望优先保留的休息日
- 排程逻辑重构：
  - 移除旧的整批优先、单日贪心入口，统一为单一排程主逻辑
  - 排程窗口从整天 `deadline` 改为分钟级窗口，按每个零件所属工序分别计算
  - 每个批次先尽量满足 `lead_time_minutes`，仅当严格窗口内仍有缺口时，才启用该批次自己的“提前半天”兜底窗口
  - 在合法窗口内优先把工作日压向 `12h / 24h`，取消原先 `18h` 一类“小夜班”目标
  - 在不破坏窗口和工时档位的前提下，尽量减少单日规格种类
  - 整班整理规则调整为：`0~11h -> 尽量补到 12h`，`11~13h -> 视为接近 12h 保持不动`，`13~23h -> 尽量补到 24h`，`23~24h -> 视为接近 24h 保持不动`
  - `<8h` 的短尾天会优先尝试并回前面已开工的天，哪怕接收日因此超过 `12h`，以便尽量多休一天
  - 若传入 `rest_days`，会优先使用非休息日排产，并尽量让每个 7 天区间保留至少半天或一天休息；在产能不足时才允许吃掉休息日
  - 移除接板时间相关逻辑，不再输出接板字段
- 响应结构调整：
  - `CoilSpecSummary` 移除 `coil_count`
  - `CoilSpecSummary` 移除 `splicing_time_minutes`
  - `pre_stock_requirements` 保持按工序天输出缺口明细，且每天只表示当天自身缺口，不做跨天累加
